# java-lotto-precourse

## 범위
- 입력
    1) 구입 금액(1,000원 단위, 양의 정수)
    2) 당첨 번호(쉼표 구분 6개, 1~45, 중복 없음)
    3) 보너스 번호(1~45, 당첨 번호와 중복 금지)
- 출력
    - `N개를 구매했습니다.` + 발행 로또 번호 오름차순
    - `당첨 통계` + `---`
    - 5등~1등 건수 및 총 수익률(소수점 1자리, 둘째 자리 반올림)

## 기능 목록
1. 구입 금액 입력 — 프롬프트 출력 후 한 줄 입력 수신
2. 구입 금액 검증 — 양의 정수 & 1,000원 단위(배수) 확인, 실패 시 `[ERROR]` 후 재입력
3. 로또 발행 — N = 금액/1000, 각 티켓 1~45 중복 없는 6개 생성
4. 발행 내역 출력 — `N개를 구매했습니다.` + 각 티켓 번호 오름차순 출력
5. 당첨 번호 입력 — `당첨 번호를 입력해 주세요.` 수신
6. 당첨 번호 검증 — 쉼표 구분 6개, 1~45, 중복 없음, 실패 시 재입력
7. 보너스 번호 입력 — `보너스 번호를 입력해 주세요.` 수신
8. 보너스 번호 검증 — 1~45, 당첨 번호와 중복 금지, 실패 시 재입력
9. 등수 규칙 정의 — `Rank`(1등~5등) 상금/라벨 및 매칭 규칙 확정
10. 결과 집계 — 각 티켓 매칭/보너스 반영해 등수 산정 및 등수별 카운트 합산
11. 통계/수익률 출력 — 등수별 건수, 총 상금, 수익률(소수점 둘째 자리 반올림 → 한 자리) 출력


## 구현 결정(Assumptions)
- 출력 포맷은 예시와 공백/개행까지 동일
- 상수는 전부 static final로 관리(가격/범위/상금/문구)
- UI(System.in/out) 로직은 단위 테스트에서 제외
